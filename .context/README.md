# Session Context Management

This folder contains session handoff files for maintaining context between work sessions.

## File Structure:
- `session-{id}_handoff.md` - Session handoff summaries (max 300 words)
- `session-{id}_active.md` - Active session context (if needed)

## Content Format:
Each handoff file contains:
- **Critical Decisions:** ≤150 words
- **Current State:** ≤100 words  
- **Next Session Instructions:** ≤50 words

## Usage:
- **Last working agent** creates handoff file at session end
- **Product Manager** validates context accuracy
- **Next session** auto-loads previous context
